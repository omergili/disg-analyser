# ⚡ Operative Klarheit – Radikale Prompt-Optimierung

## Problem: Zu narrativ und ausschweifend

**Vorher (narrativ):**
```
"Als I-Typ würde ich zunächst die Vision des Projekts kommunizieren 
und das Team begeistern. Dann würde ich regelmäßige Meetings 
einplanen, um die Motivation hochzuhalten..."
```
❌ Ich-Form
❌ Narrative Erzählweise
❌ Ausschweifend
❌ Keine klare Struktur

---

## Lösung: Operative Klarheit

**Nachher (operativ):**
```
**Kernziel:** Maximale Team-Begeisterung und Netzwerk-Aktivierung.

**Vorgehensweise:**
→ Erstelle Vision-Präsentation (max. 10 Slides)
→ Plane wöchentliche 30min Team-Updates
→ Aktiviere Netzwerk für Early Adopters
→ Setze monatliche Erfolgs-Celebrations

**Verantwortung:** Inhaber kommuniziert Vision, Team-Leads moderieren Updates

**Key-Check:** Ist die Begeisterung messbar oder nur gefühlt?
```
✅ Sachlich
✅ Strukturiert
✅ Actionable
✅ Keine Füllwörter

## Neues Prompt-Format

### Strikte 4-Punkt-Struktur

#### 1. Kernziel (1 Satz)
```
**Kernziel:** [Was ist das Hauptziel dieser Perspektive?]
```

**Beispiele:**
- D: "Schnellste Time-to-Market bei klaren Verantwortlichkeiten"
- I: "Maximale Team-Begeisterung und Netzwerk-Aktivierung"
- S: "Stabile Prozesse mit hoher Team-Zufriedenheit"
- G: "Fehlerfreie Umsetzung durch systematische Qualitätssicherung"

#### 2. Vorgehensweise (3-4 Stichpunkte)
```
**Vorgehensweise:**
• [Verb] [konkrete Aktion]
• [Verb] [konkrete Aktion]
• [Verb] [konkrete Aktion]
```

**Erlaubte Verben:**
- Erstelle, Definiere, Setze, Implementiere
- Prüfe, Validiere, Teste, Verifiziere
- Delegiere, Beauftrage, Weise zu
- Plane, Strukturiere, Organisiere
- Dokumentiere, Protokolliere, Tracke
- Kommuniziere, Informiere, Alignment
- Etabliere, Implementiere, Installiere

**Verboten:**
- "Ich würde..."
- "Man könnte..."
- "Es wäre gut..."
- Narrative Füllwörter

#### 3. Verantwortung (Wer macht was?)
```
**Verantwortung:** [Rolle/Person] [übernimmt] [was]
```

**Setting-spezifisch:**

**Solo:**
- "Inhaber selbst"
- "Externe Dienstleister für [X]"
- "Tool [Y] automatisiert [Z]"

**Kleines Team:**
- "Team-Lead koordiniert"
- "Entwickler A übernimmt Backend"
- "Designer B erstellt Mockups"

**Spezialisten:**
- "Abteilungsleiter Technik verantwortet"
- "Spezialist für Arbeitsrecht prüft"
- "Externer Consultant führt Audit durch"

#### 4. Key-Check (Kritische Frage)
```
**Key-Check:** [Eine Frage zur Selbstprüfung]
```

**Beispiele:**
- "Ist die Deadline realistisch oder Wunschdenken?"
- "Haben wir das Budget für diese Qualität?"
- "Fühlt sich das Team wirklich eingebunden?"
- "Sind die Daten validiert oder Annahmen?"

## Beispiel: Kompletter Output

### Input
```
Aufgabe: "Ein neues CRM-System einführen"
Typ: G (Gewissenhaft)
Setting: Spezialisten-Team
```

### D-Strategie
```
**Kernziel:** CRM-Launch in 8 Wochen mit klaren Meilensteinen.

**Vorgehensweise:**
→ Definiere Go-Live-Datum (fixe Deadline)
→ Setze wöchentliche Steering-Meetings (Freitag 15:00)
→ Etabliere Eskalationspfad bei Blockers
→ Implementiere tägliches 10min Standup

**Verantwortung:** Projektleiter steuert, IT-Lead implementiert, Fachbereich testet

**Key-Check:** Wer entscheidet bei Konflikten zwischen Speed und Quality?
```

### I-Strategie
```
**Kernziel:** Hohe User-Akzeptanz durch aktive Einbindung.

**Vorgehensweise:**
→ Erstelle User-Champions-Programm (5-10 Early Adopters)
→ Plane monatliche Demo-Sessions mit Feedback-Runden
→ Kommuniziere Quick Wins sichtbar (Newsletter, Town Hall)
→ Organisiere Launch-Event mit Gamification

**Verantwortung:** Marketing kommuniziert, User-Champions schulen Peers, IT supportet

**Key-Check:** Messen wir Akzeptanz oder verlassen wir uns auf Bauchgefühl?
```

### S-Strategie
```
**Kernziel:** Schrittweise Migration ohne Disruption bestehender Prozesse.

**Vorgehensweise:**
→ Plane 3-Phasen-Rollout (Pilot → Dept 1 → Full)
→ Dokumentiere alle Prozessänderungen schriftlich
→ Etabliere Support-Hotline ab Tag 1
→ Sammle wöchentliches User-Feedback strukturiert

**Verantwortung:** Change Manager koordiniert, Abteilungsleiter betreuen Teams, IT-Support 1st Level

**Key-Check:** Haben wir genug Buffer für unvorhergesehene Probleme?
```

### G-Strategie
```
**Kernziel:** Fehlerfreie Implementation durch systematische Qualitätssicherung.

**Vorgehensweise:**
→ Erstelle detaillierte Anforderungsspezifikation (unterschrieben)
→ Implementiere 3-stufiges Test-Protokoll (Unit → Integration → UAT)
→ Dokumentiere alle Customizations und Schnittstellen
→ Validiere Datenmigration mit Stichproben (10% Manual Check)

**Verantwortung:** QA-Manager verantwortet Tests, Datenbankadmin migriert Daten, Compliance prüft DSGVO

**Key-Check:** Sind alle Edge Cases getestet oder nur Happy Path?
```

## Prompt-Verbesserungen

### Neue Regeln im Prompt

```javascript
REGELN:
- KEINE Ich-Form ("Ich würde...")
- KEINE narrativen Einleitungen
- NUR operative Handlungsanweisungen
- Verben am Satzanfang
- KEINE Füllwörter
- Sachlich wie ein Projektmanager-Assistent
- Kurz und präzise
```

### Setting-Kontext präzisiert

**Vorher:**
```
focus: 'Selbstmanagement, Disziplin, Zeitersparnis'
```

**Nachher:**
```
roleContext: 'Inhaber/Freelancer selbst'
delegation: 'Externe Dienstleister oder Tools'
```

→ Klarer Bezug zu WER die Aufgaben übernimmt

## Qualitätsmerkmale

### ✅ Operative Klarheit
- Jeder Punkt ist eine Handlung
- Beginnt mit Verb
- Konkret und messbar

### ✅ Verantwortung definiert
- Klar wer was macht
- Setting-spezifisch (Solo/Team/Spezialisten)
- Keine Unklarheiten

### ✅ Kritische Reflexion
- Key-Check zwingt zu Selbstprüfung
- Deckt Schwachstellen auf
- Fördert Realismus

### ✅ Scanbarkeit
- Struktur immer gleich
- Visuell getrennt
- Schnell erfassbar
